# 採用サイト要件定義書

## 1. サイトの目的

株式会社MedicAI様の採用活動を促進するため、企業の魅力と先進性を的確に伝え、求める人物像に合致する優秀な人材（エンジニア、プロダクトマネージャーなど）からの応募を最大化する、モダンで洗練された採用サイトを構築する。

## 2. デザインコンセプト

    -   **テーマ**: 「テクノロジーで、医療の未来を創造する」
    -   **デザインの方向性**:
    -   **先進性と信頼性**: AIやクラウドといった先進技術を扱う企業としての技術力の高さと、医療という領域が持つ信頼性・クリーンさを両立させる。
    -   **ミニマル & モダン**: 白を基調としたクリーンなレイアウトに、高品質な写真や洗練されたタイポグラフィを配置。情報は詰め込みすぎず、余白を活かしたデザインで訴求力を高める。
    -   **カラー**: ベースカラーはホワイト(#FFFFFF)。メインカラーに信頼感を抱かせるブルー、アクセントカラーに生命感や健康を想起させるグリーンを使用し、先進的かつ落ち着いた印象を与える。
    -   **インタラクション**: スクロールに応じた要素の表示や、マウスオーバー時のエフェクトなど、ユーザー体験を向上させるための、控えめで心地よいアニメーションを導入する。

## 3. サイト構成案（サイトマップ）

| ページ名 | URL | 概要 |
| :--- | :--- | :--- |
| **トップページ** | `/` | サイトの入り口。企業のミッションとビジョンを伝え、各コンテンツへ誘導する。 |
| **ミッション** | `/mission` | 代表メッセージと共に、ミッションとビジョンをより深く伝える。 |
| **事業内容** | `/services` | 提供するSaaSやプラットフォームについて、図や画像を交えて分かりやすく紹介。 |
| **カルチャー** | `/culture` | 働く環境、求める人物像、社員インタビューなどを通して、企業文化を伝える。 |
| **募集職種一覧** | `/careers` | 現在募集中の職種を一覧で表示。 |
| **募集職種詳細** | `/careers/{id}` | 各職種の具体的な仕事内容、応募要項、やりがいなどを掲載。 |
| **ブログ** | `/blog` | 技術情報の発信や社内イベントの紹介など、企業の「今」を伝える。 |

## 4. コンテンツ要件

### 既存情報の活用 (`corp_info.md`より)

-   企業名、ミッション、ビジョン
-   事業内容、提供サービス
-   求める人物像
-   募集職種

### 新規で必要となるコンテンツ

-   **写真・動画素材**:
    -   トップページで使用する、先進的な医療やチームの協業をイメージさせる高品質な動画または画像。
    -   オフィス環境がわかる写真（執務エリア、会議室、リフレッシュスペースなど）。
    -   社員インタビューに掲載する、社員の顔写真や仕事風景。
-   **テキストコンテンツ**:
    -   代表メッセージ。
    -   各事業内容のより詳細な説明。
    -   社員インタビュー記事（各職種最低1名）。
    -   福利厚生や研修制度の具体的な内容。
    -   各募集職種の詳細な要項（職務内容、必須・歓迎スキル、チーム構成、想定年収など）。
    -   ブログ記事。

## 5. 技術スタック提案

モダンな開発環境と優れたパフォーマンスを実現するため、以下の技術スタックを提案します。

-   **フロントエンド**: **Next.js (App Router)**
    -   理由: 高いパフォーマンス（SSR/SSG）、優れたSEO対策、効率的な開発体験を提供。
-   **スタイリング**: **Tailwind CSS v4**
    -   理由: ユーティリティファーストな設計。v4の採用によりビルドパフォーマンスとCSS変数の扱いが向上。
-   **アニメーション**: **Framer Motion** (新規追加)
    -   理由: 「超モダン」なUXを実現するため。スクロールアニメーションやページ遷移の滑らかさを担保し、先進性を表現する。
-   **CMS (コンテンツ管理)**: **Headless CMS architecture (Dev: Mock CMS)**
    -   理由: 本番環境ではContentful等を想定するが、開発フェーズではローカルのJSON/TypeScript定数によるMock CMSを採用し、外部依存を排除して高速にUI/UXを作り込む。
-   **ホスティング**: **Vercel**
    -   理由: Next.jsとの親和性が非常に高く、自動デプロイなど開発から公開までのフローがスムーズ。
